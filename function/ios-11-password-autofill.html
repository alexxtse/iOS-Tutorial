iOS 11 自动填充密码

iOS 11 自动填充密码

原文地址 / 作者：Anil Varghese

翻译于：2017/08/25，Alex Tse

手动输入密码是阻碍用户体验的根源之一，而 iOS 11 带来了自动填充密码功能，用户无须记住密码，就像 Safari 建议密码功能一样，App 也能自动填写用户凭证，给予了用户更快捷的登录方式。

如何实现？

要在你的 App 实现这个功能非常的简单，只需要在 username 和 password 的文本框(textfield)设置 textContentType 属性。

userNameTextField.textContentType = .username
passwordTextField.textContentType = .password

当然也可以在 storyboard 上设置。

QuickType bar

运行 App，看看屏幕上会出现什么。当点击 username 或 password 的文本框，QuickType bar 会带着一个钥匙图标出现。点击它会要求你使用 Touch ID 进行身份验证，然后列出在钥匙串(keychain)保存的所有密码，选择其中一个，将会自动填入 username 和 password 的文本框。

关联 App 和网站

到目前为止，App 还不知道我们关联的是哪个网站，所有他一次性把所有的密码信息显示出来。接下来我们要把 App 和网站关联起来。首先，我们需要在 Xcode 上的 Associated Domains(关联域，project settings-> Capabilities -> Associated Domains) 添加网站域。

App 知道关联哪个网站了，现在网站需要确认关联。创建一个名为 apple-app-site-association 的 JSON 文件，内容如下：

{
   “webcredentials”: {
         “apps”: [“SR9HR505U.com.anilvarghese.pw-autofill”]
    }
}

SR9HR505U 是你的 Team ID，com.anilvarghese.pw-autofill 是 App 的 Bundle ID。将文件放到网站根目录，或者放到 .well-known 文件夹（首选此方式）。

App 运行时便会使用安全连接在服务器上找这个文件（服务器必须设置有 SSL）。App 自动向 https://www.example.com/apple-app-site-association 发送请求，响应之后查找 App 的 Bundle ID。

运行 App 之后，你可以通过查看控制台的设备日志来检查设置是否正确。如果一切 OK，则会看到 "SiteApproved"。

现在 App 和网站已经双向关联了。运行 App，你会注意到 QuickType bar 已经正确的提示对应的凭证，你只需点击一下即可。

酷 😎 ！试一下吧，如果有任何问题请在评论中告诉我们。

分享网络凭证（更新）

有些人问自动填充功能是否在钥匙扣里自动保存/更新密码？

它不会自动更新密码，但使用 共享网络凭证(shared web credential) 可以让原生 iOS App 和网站进行共享凭证。用户在 App 里创建一个新账号，修改密码，或者删除账号，都会通过 iCloud 进行同步，让 Safari 与这些信息保存一致，从而达到 App 与网站之间共享凭证。当然，这些操作还是需要得到用户同意。

苹果的这篇 Managing shared credentials(管理共享凭据) 文章很好的解释了共享网络凭证。
