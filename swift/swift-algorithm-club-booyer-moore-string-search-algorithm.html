朴素模式！

朴素模式(brute force)是相对直观的方法，要了解这个搜索模式，可看看下面的 "HELLO WORLD"：

按本教程的计划，我们做以下几点约定：

  1. 搜索算法区分大小写。
  2. 算法返回第一个匹配的索引(index)。
  3. 可以部分匹配。例如：
let text = "HELLO WORLD"
text.index(of: "ELLO") // returns 1
text.index(of: "LD") // returns 9

这个算法相当的直接，假设要查找 "LO"，查找的字符串会循环与原字符串做比较，一旦首字符与原字符串的一个字符匹配上，接下来就尝试匹配下一个字符，如果下一个字符无法匹配，将移动到下一个位置继续比较：

实现

使用 Xcode 9 beta 2 或更高版本，创建一个 Swift playground，删掉 playground 准备好的样例代码，写一个 String 的 extension(扩展) 方法：

extension String {
  func index(of pattern: String) -> Index? {
    // more to come
    return nil
  }
}

这个方法要做的事很简单：给定一个字符串（称为源字符串），检查是否包含其他字符串（称为 pattern 字符串），如果可以匹配的上，则返回匹配的第一个字符的索引，如果匹配不上，则返回 nil。

Swift 4 为 String 加入了 indices 属性，里面包含了所有字符的索引下标，可用于循环遍历源字符串。现在用下面的代码更新方法：

func index(of pattern: String) -> Index? {
  // 1
  for i in indices {

    // 2
    var j = i
    var found = true
    for p in pattern.indices {
      guard j != endIndex && self[j] == pattern[p] else { found = false; break }
      j = index(after: j)
    }
    if found {
      return i
    }
  }
  return nil
}

这正是我们需要的：

  1. 循环遍历源字符串的所有字符索引。
  2. pattern 字符串字符与源字符串字符进行匹配。

如果匹配上了，将会返回索引。现在可以开始在 playground 进行测试，添加下面代码：

let text = "Hello World"
text.index(of: "lo") // returns 3
text.index(of: "ld") // returns 9

算法正常运行，但效率相对较低。在下一节，你将会看到如何使用一个巧妙的方法来优化算法。

Boyer Moore 字符串搜索

其实在匹配的时候不需要每一个字符都进行对比，是可以一次跳过多个字符的。这个跳跃式的算法叫 Boyer Moore，已经诞生了有一段时间了，它被认为是所有字符串搜索算法的基准。

这个算法与朴素模式相比，有两个不同的关键点：

  1. 匹配模式是向后匹配的。
  2. 在遍历过程中使用跳跃表进行跳跃。

func index(of pattern: String) -> Index? {
  return nil 
}

跳跃表

先定义一个跳跃表，在 String 扩展里面添加下面代码：

fileprivate var skipTable: [Character: Int] {
  var skipTable: [Character: Int] = [:]
  for (i, c) in enumerated() {
    skipTable[c] = count - i - 1
  }
  return skipTable
}

这里将字符串进行枚举，并返回一个字典，字符串的字符作为字典的 key，对应的数字则表示为跳过的次数。在 playground 底部添加下面的代码进行测试：

let helloText = "Hello"
helloText.skipTable.forEach { print($0) }

你会在控制台看到下面的内容：

(key: "H", value: 4)
(key: "L", value: 1)
(key: "O", value: 0)
(key: "E", value: 3)

这个就是之前所看到的跳跃表。

匹配

Boyer Moore 算法的另一个部分就是它是向后匹配的，你需要写一个方法来实现它，这个方法有 3 个目的：

  1. 向后匹配字符。
  2. 任意一个字符匹配失败，返回 nil。
  3. 如果全部匹配成功，返回第一个字符与源字符串匹配的 String.Index。

在 skipTable 方法下写上下面的代码：

// 1
fileprivate func match(from currentIndex: Index, with pattern: String) -> Index? {
  // more to come

  // 2
  return match(from: index(before: currentIndex), with: "\(pattern.dropLast())")
}

这个递归方法实现字符串的字符逐一向后匹配：

  1. currentIndex 跟踪源字符串当前需要匹配的字符。
  2. 每一次递归都需要将索引减一，并且去掉 pattern 字符串的最后一个字符。

这个方法的执行过程如下图所示：

现在是时候来处理比较的逻辑了，更新 match 方法：

fileprivate func match(from currentIndex: Index, with pattern: String) -> Index? {
  // 1
  if currentIndex < startIndex { return nil }
  if currentIndex >= endIndex { return nil }

  // 2
  if self[currentIndex] != pattern.last { return nil }

  // 3
  if pattern.count == 1 && self[currentIndex] == pattern.last { return currentIndex }
  return match(from: index(before: currentIndex), with: "\(pattern.dropLast())")
}
  1. 这里是越界的检查，如果 currentIndex 越界了，则返回 nil。
  2. 如果字符不匹配，同样返回 nil。
  3. 如果 pattern 字符串最后一个字符匹配成功了，则返回当前索引(currentIndex)，这同样也是表示 pattern 字符串第一个字符首次匹配的地方。

为了能更清楚的说明过程，我将多个逻辑判断分开成多个语句，你可以使用 guard 这种更加简洁的方式来重新它：

guard currentIndex >= startIndex && currentIndex < endIndex && pattern.last == self[currentIndex] 
  else { return nil }
if pattern.count == 1 && self[currentIndex] == pattern.first { return currentIndex }

跳跃表和匹配方法都已经准备好了，是时候解决最后一部分难题了。

index(of:)

更新 index 方法：

func index(of pattern: String) -> Index? {
  // 1
  let patternLength = pattern.count
  guard patternLength > 0, patternLength <= count else { return nil }

  // 2
  let skipTable = pattern.skipTable
  let lastChar = pattern.last!

  // 3
  var i = index(startIndex, offsetBy: patternLength - 1)

  // more to come...
  return nil
}

这里是一些基础准备：

  1. 首先检查 pattern 字符串的长度是否在源字符串长度的范围之内。
  2. 获得 pattern 字符串跳跃表，和它的最后一个字符。
  3. 初始化 String.Index 用于循环遍历。由于是从字符串向后匹配，所以索引根据 pattern 字符串的长度来偏移到需要的位置。

接下来要定义匹配和遍历的逻辑，将下面代码添加到 return 语句之前：

// 1
while i < endIndex {
  let c = self[i]

  // 2
  if c == lastChar {
    if let k = match(from: i, with: pattern) { return k }
    i = index(after: i)
  } else { 
    // 3
    i = index(i, offsetBy: skipTable[c] ?? patternLength, limitedBy: endIndex) ?? endIndex
  }
}

这里需要详细的说明：

  1. 你将不断的遍历源字符串，直到大于 endIndex。
  2. 如果源字符串的当前字符与 pattern 字符串最后一个字符匹配上了，则运行 match 方法。如果不是返回 nil，说明源字符串包含 pattern 字符串，同时返回相应的索引。如果返回的是 nil，则跳到下一个索引。
  3. 如果源字符串的当前字符与 pattern 字符串最后一个字符不匹配，则应该查看跳跃表，看看当前索引的位置能一次跳过几个。如果跳过了源字符串的长度，则直接取最后字符位置的索引(endIndex)。

把下面代码添加到 playground 底部进行测试：

let sourceString = "Hello World!"
let pattern = "World"
sourceString.index(of: pattern)

你应该会得到一个 6 的索引，哇喔！一切正常！
