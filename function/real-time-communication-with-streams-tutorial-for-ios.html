iOS 实时流通讯

iOS 实时流通讯

原文地址 / 作者：Luke Parham

翻译于：2017/08/11，Alex Tse

教程由 Luke Parham 更新支持 Xcode 9 beta / iOS 11 / Swift 4 
原教程是 Cesare Rocchi 编写

自古以来，人们就已经希望能有更好的沟通方式。从信鸽到无线电波，我们一直都在努力寻找更便捷有效的信息传达。

时代发展，如今我们的信息传达都依赖着一种必不可少的技术：network socket。

Sockets 存在现在的网络基础设施的 layer 4 (传输层)，从发短信到在线游戏，是任何在线通信的核心。

为什么是 Sockets？

你可能会惊奇的问，"为什么我们需要一个比 URLSession 更低层的东西？"如果你没有惊讶，我就当做你有吧。

好问题！URLSession 通信是基于HTTP网络协议的。HTTP的通信方式是在一个「请求 - 响应」中发生的。这意味着大多数的 App 都遵循着大体相同的网络模式：

  1. 从服务器请求一些 JSON 数据。
  2. 在回调或委托函数中接收和使用这些 JSON 数据。

但是，当你想服务器能主动告诉 App 一些内容，应该怎么办？这样做其实并没有很好的映射到HTTP。当然，你可以通过持续的ping服务器看看有没有内容更新来达到你的目的，这种方式称为轮询，或者你可以耍点小聪明使用长轮询的技术。但使用这些技术都有点不自然，并且它们都有自身的陷阱。一天结束了，如果你发现这不是工作上正确的工具，为什么还要限制自己使用「请求 - 响应」的方式？

在这个有关流的教程里，你将学习如何低抽象层，使用 Sockets 直接创建一个即时聊天室应用程序。

Sockets 不需要让每个客户端都去实时检查服务器是否有新消息，它将在聊天会话持续期间仍然保持打开输入和输出流。

入门

开始之前，先下载一些准备材料，里面包含了一个聊天室 App 和用 Go 写的一个简单服务。别担心，我们不需要你写任何 Go 代码，为了编写客户端你只需要把服务启动运行就可以了。

让服务跑起来

服务是由 Go 编写的，同时也已经编译好了。如果你不信任网络上任何已经编译好的可执行文件，那么我也包含了源文件，你可以自行编译。

要运行预编译服务，打开终端，导航到刚刚下载的准备材料目录，并输入下面的命令，接着会提示你输入密码：

sudo ./server

在输入密码之后，你应该会看到 127.0.0.1:80 被监听。现在你的聊天服务已经准备好了！你可以直接看下一章节了。

如果你想自己编译服务，你需要通过 Homebrew 来安装 Go。

如果你没有 Homebrew，那么你先要安装他。打开终端，把下面这段命令输入粘贴进去：

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

然后使用这个命令安装 Go：

brew install go

完成之后，就导航到准备材料的目录，并使用 build 命令来构建服务。

go build server.go

最后，你可以使用本节开头列出的命令来启动你的服务。

看看现有的应用程序

下一步，打开 DogeChat 项目并运行，看看为你准备的内容。

如上图所示，目前 DogeChat 可以让用户输入用户名，并进入聊天室。可惜的是编写这个 App 的人真的不知道如何实现聊天功能，所以他们只实现了所有的 UI 和基础通道，剩下的让你来实现实际的网络层。

创建一个聊天室

现在开始编写代码，找到 ChatRoomViewController.swift。你可以看到已经为你准备好了一个 view controller，一个能够获得输入栏里的字符串信息，以及用于显示信息的 tableview，cell 是通过 Message 对象来自定义的。

既然已经有了一个 ChatRoomViewController，那么再创建一个 ChatRoom 类就行了。

在为这个类编码之前，我想快速的列出一个职责清单。对这个类，我们希望能够处理：

  1. 打开聊天室服务连接
  2. 允许用户通过提供用户名就可加入到聊天室
  3. 允许用户发送和接收信息
  4. 在完成聊天之后可关闭连接

现在你已经知道要做什么了，按 ⌘+n 新建一个文件，选择 Cocoa Touch Class 并命名为 ChatRoom。

创建输入输出流

接下来用下面的代码替换文件中的代码：

import UIKit
class ChatRoom: NSObject {
  //1
  var inputStream: InputStream!
  var outputStream: OutputStream!
  //2
  var username = ""
  //3
  let maxReadLength = 4096
}

在这里，你定义了 ChatRoom 类并声明了一些属性，以便有效地进行通信。

  1. 首先，你已经有了输入和输出流了，使用这对类可以让应用程序与聊天服务之间建立一个基于 Socket 的连接。
  2. 下一步，你有一个 username 的变量，用于存储当前用户的用户名。
  3. 最后，你有一个 maxReadLength，这个变量是控制单条信息最多能发送多少数据。

然后回到 ChatRoomViewController.swift，在上面的属性区域里添加一个聊天室属性。

let chatRoom = ChatRoom()

现在你已经为你的类建立了基础结构，是时候实现你职责清单上的第一件事：打开应用程序和服务之间的连接。

打开连接

返回到 ChatRoom.swift，在定义属性的下方添加如下代码：

func setupNetworkCommunication() {
  // 1
  var readStream: Unmanaged<CFReadStream>?
  var writeStream: Unmanaged<CFWriteStream>?

  // 2
  CFStreamCreatePairWithSocketToHost(kCFAllocatorDefault,
                                     "localhost" as CFString,
                                     80,
                                     &readStream,
                                     &writeStream)
}

这里看看代码发生了什么：

  1. 这里设置了两个未初始化且不会自动进行内存管理的 Socket 流。
  2. 然后把读和写的 Socket 流绑定在一起，并连接到端口为80的 Socket 主机上。
    这个函数需要四个参数。第一个参数是初始化流使用的分配器（allocator）类型。你应该尽可能的使用 kCFAllocatorDefault，当然如果你遇到了其他的情况，需要进行特殊处理，也可以使用其他的方式。
    下一个参数是指定主机名（hostname）。在这个范例中，只需要连接到本地即可，如果你有远程服务器的特定 IP，同样能在这里使用。
    再下一个参数是指定你是通过80端口连接的，这是我们设置可监听的服务器端口。
    最后是传入读和写的流的指针，因此在连接读和写的流时，可在函数内部进行创建并初始化。

现在你已经初始化了流，添加下面代码来存储它们的 Retained 引用：

inputStream = readStream!.takeRetainedValue()
outputStream = writeStream!.takeRetainedValue()

调用 Unmanaged 对象的 takeRetainedValue() 允许你获得一个被保留的引用同时销毁不稳定的引用，确保后面不会有内存泄露。现在你可以在需要的时候使用输入和输出流。

接下来为了让你的 App 能正确的响应网络事件，这些流需要添加到一个 run loop 中。在 setupNetworkCommunication 末尾添加下面两行代码来实现：

inputStream.schedule(in: .current, forMode: .commonModes)
outputStream.schedule(in: .current, forMode: .commonModes)

你终于可以准备打开洪水闸门了！让派对开始吧！（再次在 setupNetworkCommunication 底部添加下面代码）

inputStream.open()
outputStream.open()

这就是全部了，完成之后切换到 ChatRoomViewController.swift，把下面的一行代码添加到 viewWillAppear(_:) 方法中：

chatRoom.setupNetworkCommunication()

现在你的 App 已经和本地服务之间建立了一个连接。你也可以继续的构建和运行你的 App，但不会看到你想要的效果，因为你还没真正的在这个连接上做任何事情。

加入聊天

既然已经和服务建立了连接，是时候开始做聊天的事情了。首先你会说聊天的人是谁，然后那个人还要向其他人发送消息。

这里提到了重要的一点：由于有两种消息，所以你需要想到一个可以区分它们的方式。

通信协议

降到 TCP 层有一个好处就是能够自定义"协议"来确认消息是否有效。在 HTTP 中你需要想那些令人讨厌的动词（如 GET，PUT 和 PATCH），还需要构建 URL，设置合适头信息和其他各种东西。

这里我们只是有两种类型的消息。所以你可以发送：

iam:Luke

进入聊天室之后，向他们告诉你是谁。

然后你可以说：

msg:Hey, how goes it mang?

向聊天室其他人发送消息。

这个发送行为纯粹，简单。

但这也明显是不安全的，不能在工作中使用。

服务器需要有一个期望值，你可以在 ChatRoom 上编写一个允许用户进入聊天室的方法，而唯一的参数是用户名。

在你刚刚实现的设置方法的下面，添加如下代码来实现这个功能：

func joinChat(username: String) {
  //1
  let data = "iam:\(username)".data(using: .ascii)!
  //2
  self.username = username

  //3
  _ = data.withUnsafeBytes { outputStream.write($0, maxLength: data.count) }
}
  1. 使用一个简单的聊天室协议构建消息。
  2. 保存传入的用户名，以便后面发送消息的时候使用它。
  3. 将消息写入到输出流，这可能比你想的要复杂一些，write(_:maxLength:) 方法引用一个不安全的字节指针作为第一个参数。withUnsafeBytes(of:_:) 方法为你提供了一个便捷的处理方式将一些不安全的数据指针控制在一个安全的范围内。

现在方法准备好了，在 ChatRoomViewController.swift 将加入聊天室的代码添加到 viewWillAppear(_:) 方法里的下面:

chatRoom.joinChat(username: username)

运行，输入你的名字，然后，点击进入看看...

怎么效果一样的？！

莫慌，我来解释一下。去看看终端，如果 127.0.0.1:80 处于正常的监听下，你应该会看到 Luke 已加入，或者你取了其他的名字，反正大概会看到类似的信息。

这是个好消息，但你应该更希望能在手机屏幕上看到一些成功的信息。

响应传来的消息

幸运的是服务器现在可以接收你发来的消息，例如刚刚加入聊天室的消息，接着就是向聊天室的所有人发送聊天内容了。ChatRoomViewController 的 tableview 的 cell，已经设置并准备好显示任何传来的消息。

你所需要做的就是使用 inputStream 来捕获传来的消息，并将它们转换成 Message 对象，然后传给 tableview 来显示。

为了响应传来的消息，你需要为聊天室加入一个输入流的委托，在 ChatRoom.swift 下面添加一个扩展：

extension ChatRoom: StreamDelegate {

}

现在已经加入 StreamDelegate 协议，可以声明 inputStream 的委托方法。

回到上面的 setupNetworkCommunication()，在 schedule(_:forMode:) 前面，添加下面代码。

inputStream.delegate = self

下一步，将 stream(_:handle:) 的实现添加到扩展里。

func stream(_ aStream: Stream, handle eventCode: Stream.Event) {
    switch eventCode {
    case Stream.Event.hasBytesAvailable:
      print("new message received")
    case Stream.Event.endEncountered:
      print("new message received")
    case Stream.Event.errorOccurred:
      print("error occurred")
    case Stream.Event.hasSpaceAvailable:
      print("has space available")
    default:
      print("some other event...")
      break
    }
}

这里是处理一些传来的相关流可能会发生的事件。你可能真正感兴趣的是 Stream.Event.hasBytesAvailable，因为这个代表有一个传入的消息等待被读取。

接下来你要写一个方法来处理传来的消息，在这个方法下面添加：

private func readAvailableBytes(stream: InputStream) {
  //1
  let buffer = UnsafeMutablePointer<UInt8>.allocate(capacity: maxReadLength)

  //2
  while stream.hasBytesAvailable {
    //3
    let numberOfBytesRead = inputStream.read(buffer, maxLength: maxReadLength)

    //4
    if numberOfBytesRead < 0 {
      if let _ = stream.streamError {
        break
      }
    }

    //Construct the Message object

  }
}
  1. 设置一个最大可接收的字节长度为 maxReadLength 的 buffer。
  2. 循环读取的输入流字节。
  3. read(_:maxLength:) 方法从流中读取字节，并传入 buffer。
  4. 如果读取的结果返回一个负数，则需要处理一下错误。

在输入流有可读取字节时才需要调用此方法，所以回到 stream(_:handle:) 方法的 switch 语句，在 case Stream.Event.hasBytesAvailable 中的 print 语句下面执行这个方法。

readAvailableBytes(stream: aStream as! InputStream)

到这里你那甜蜜的 buffer 已经充满了字节！在完成此方法之前，你需要写另一个方法来帮助你将 buffer 转换成 Message 对象。

将下面方法放到 readAvailableBytes(_:) 下面。

private func processedMessageString(buffer: UnsafeMutablePointer<UInt8>,
                                    length: Int) -> Message? {
  //1
  guard let stringArray = String(bytesNoCopy: buffer,
                                 length: length,
                                 encoding: .ascii,
                                 freeWhenDone: true)?.components(separatedBy: ":"),
    let name = stringArray.first,
    let message = stringArray.last else {
      return nil
  }
  //2
  let messageSender:MessageSender = (name == self.username) ? .ourself : .someoneElse
  //3
  return Message(message: message, messageSender: messageSender, username: name)
}
  1. 通过传入 buffer 和长度初始化一个 String。
    让文本用 ASCII 编码，String 初始化完成并释放 buffer。然后通过 : 字符将传入的消息拆分，这样就可以分别获得发送人和消息内容的文本。
  2. 基于名字弄清楚消息是你发的还是其他人发的。要注意，在一个真实应用的 App 是不会使用这种方式，他们会用类似用户唯一 token 进行区分，但在这个案例中，这样做已经可以满足我们需求了。
  3. 最后实例一个 Message 对象。

在使用构建 Message 对象的方法前加上 if-let，并插入到 readAvailableBytes(_:) 末尾。

if let message = processedMessageString(buffer: buffer, length: numberOfBytesRead) {
  //Notify interested parties

}

现在可以发消息给人了，但是发给谁呢？

创建 ChatRoomDelegate 协议

你已经可以告诉 ChatRoomViewController.swift 有新消息了，但你不需要引用它。由于它持有 ChatRoom 的强引用，你不想显性创建一个循环依赖的 ChatRoomViewController 属性。

是时候设置委托协议了。ChatRoom 只负责告诉谁有新消息，至于是谁它不关心。

到 ChatRoom.swift 顶部，添加一个简单的协议。

protocol ChatRoomDelegate: class {
  func receivedMessage(message: Message)
}

添加一个可选的弱属性来保存是谁对 ChatRoom 委托的引用。

weak var delegate: ChatRoomDelegate?

现在你可以真正的完成 readAvailableBytes(_:) 方法了，将下面代码添加到 if-let 语句里面。

delegate?.receivedMessage(message: message)

准备工作完成，回到 ChatRoomViewController.swift 在 MessageInputDelegate 下面添加添加一个遵循这个协议的扩展。

extension ChatRoomViewController: ChatRoomDelegate {
  func receivedMessage(message: Message) {
    insertNewMessageCell(message)
  }
}

正如我之前所说，显示内容的事已经帮你准备好了，所以 insertNewMessageCell(_:) 接收到你传入的消息就会用合适的 cell 把消息添加到 tableview 上。

现在将 view controller 分配给 chatRoom 委托，在 viewWillAppear(_:) 里添加下面一行代码。

chatRoom.delegate = self

再运行一次，输入名字，点击进入。

🎉 聊天室现在正确的显示你进入了房间，你已经成功的向基于 Socket 的 TCP 服务发送和接收信息。

发送消息

现在 ChatRoom 可以正常的发送和接收信息，是时候让用户来回发送真实的内容了。

回到 ChatRoom.swift，将下面方法添加到类定义里的底部。

func sendMessage(message: String) {
  let data = "msg:\(message)".data(using: .ascii)!

  _ = data.withUnsafeBytes { outputStream.write($0, maxLength: data.count) }
}

这个方法有点像之前写的 joinChat(_:)，其中 msg 后面跟着是你发送的实际文本。

由于你希望通过 inputBar 来告诉 ChatRoomViewController 用户已经点击 Send 发送了信息，所以回到 ChatRoomViewController.swift 并找到 MessageInputDelegate 扩展。

这里你会看到一个 sendWasTapped(_:) 的空方法，在需要的时候会被执行。只需把要发送的消息传给 chatRoom 就可以了。

chatRoom.sendMessage(message: message)

这就搞定了！服务器会接收信息，然后转发给所有人，ChatRoom 将会和进入聊天室的方式一样处理新的消息。

现在运行下程序，输入点东西看看。

如果你想看别人在聊天室回你消息，你可以开一个新的终端窗口，输入下面命令：

telnet localhost 80

这个命令让你连接到 TCP 服务，现在你可以在这里发送和 App 相同的内容。

iam:gregg

然后发送一条消息

msg:Ay mang, wut's good?

恭喜，太棒了！你已经成功的开发出一个聊天 App 了。

清除会话

如果你以前有做过文件编程，那么你应该会知道当文件完成了它们的工作时，最好要关闭它。事实证明，像在 unix 的其他东西一样，打开一个 Socket 连接是通过文件句柄表示，这意味着当你使用完之后要关闭它，这像是处理其他文件一样。

因此，要在 sendMessage(_:) 后面添加下面方法。 

func stopChatSession() {
  inputStream.close()
  outputStream.close()
}

正如你所想到的，关闭流之后就无法发送和接收消息。同时也会从 run loop 中移除流。

完成这些之后，在 switch 语句的 Stream.Event.endEncountered case 添加这个方法。

stopChatSession()

然后回到 ChatRoomViewController.swift，在 viewWillDisappear(_:) 中添加同样的方法。

stopChatSession()

到这里你完成了所有的事情。祝贺你！（Profectu tuo laetamur!）👏

现在你已经掌握了（至少从这个简单的例子开始）基于 Sockets 的网络连接，下面有几个知识点可以扩充你的认识。

UDP Sockets

这个流教程是使用 TCP 通信的例子，打开连接之后并尽可能的保证数据包能够正常传输。另外，你也可以使用 UDP 或 datagram sockets 进行通信。但这些方式并没有太大的稳定保证，不过这也意味着它们的传输速度更快，开销更小。它们对类似游戏这样的应用很有帮助。如果有数据过滞后或者接收不到的经历，这说明你的连接出现了问题，本来应该接收的 UDP 数据包却被丢弃了。

WebSockets

还有一种叫 WebSockets 的技术可以使用。与传统的 TCP Sockets 不同的是它至少保持了和 HTTP 的关系，同时又可以实现即时通讯的功能，这些都是来自浏览器提供的支持和安全保证。当然 WebSockets 也是可以用在 iOS app 上的，如果有兴趣想学习更多，我们已经准备好了一个教程。
